# FastMVP

**Template** monorepo MVP factory. Supports **N backends + N frontends**. Agents turn ideas into shippable products. Not all apps exist by default — agents create what the project needs.

## Build & Run

```bash
npm install                    # install all workspaces
npm run dev                    # API: Fastify on http://localhost:3000
npm run dev:web                # Web: Expo on http://localhost:8081
npm run openapi:lint           # validate OpenAPI contracts (from apps/api)
```

Python services (when present):
```bash
cd apps/api-<name> && pip install -r requirements.txt && python -m src.main
```

## Architecture

- **Backend**: Clean Architecture + DDD + DI (TypeScript/Fastify default, Python/Flask for ML/AI)
- **Frontend**: Feature-Sliced Design (React Native Expo default, Next.js for web-only)
- **Contracts**: OpenAPI 3.1 is the single source of truth between API and frontend

## Monorepo Layout

```
apps/api/          → TypeScript backend template (created per project)
apps/api-<name>/   → Python backend (only when different stack needed)
apps/web/          → Frontend template (Expo or Next.js, created per project)
apps/mobile/       → Mobile frontend (when platform=mobile or both)
contracts/         → openapi.yaml + db/*.sql
docs/              → spec.md, tasks-*.md, figma.md (generated by agents)
resources/         → brand/, architecture/, figma/
.claude/           → agents, skills, commands, rules, learnings, hooks
.github/           → Copilot agents (same pipeline, different runtime)
```

## Non-negotiables

<important if="editing any backend code">
- OpenAPI is the boundary: no invented endpoints. Update contracts/ first, then implement.
- DI wiring ONLY in container.{ts,py}. No global singletons.
- Dependency direction: interfaces/infrastructure → application → domain. No violations.
- DB: SQL files in contracts/db/ for manual execution. No migrations, no ORM.
</important>

<important if="editing any frontend code">
- Feature-Sliced layers: app/pages/widgets/features/entities/shared
- API calls ONLY through shared/lib/api/*. No fetch/axios in pages/widgets/shared/ui.
- Use tokens from app/theme/tokens.ts. No hardcoded colors/spacing.
</important>

<important if="adding a new feature">
- Always follow vertical slices: contract → domain → application → infrastructure → interfaces → wire DI
- Append to existing spec/tasks. Never rewrite existing sections.
</important>

## Agent Pipeline

```
Idea → Specifier → Contract → [API Planner + Designer] → API Dev → Web Planner → Web Dev → Reviewer
```

Orchestrators: `fastmvp`, `feature-builder`, `bug-fixer`
Workers: `specifier`, `contract`, `api-planner`, `api-dev`, `designer`, `web-planner`, `web-dev`, `reviewer`, `retro`
