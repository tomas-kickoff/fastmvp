# fastmvp

**Template** monorepo MVP factory. You interact with AI agents, the flow refines your idea, and generates a complete app with **N backends + N frontends** automatically.

## What it does

- Starts from a short idea and turns it into a clear specification using **AI Agents**
- Defines API contracts with OpenAPI as the single source of truth
- Implements backend(s) following Clean Architecture + DDD + DI
- Implements frontend(s) following Feature-Sliced Design
- Runs a retrospective after each pipeline to continuously improve

## Dual-mode: Copilot + Claude Code

FastMVP works with **both** GitHub Copilot and Claude Code. Same pipeline, same agents.

### GitHub Copilot (VS Code)
Select an agent from the Copilot dropdown: `@FastMVP`, `@Feature Builder`, `@Bug Fixer`.

### Claude Code (CLI / IDE)
Use slash commands or mention agents directly:

| Action | Command |
|--------|---------|
| Build new MVP | `/build-mvp <idea>` |
| Add feature | `/add-feature <description>` |
| Fix bug | `/fix-bug <description>` |
| Review alignment | `/review` |
| Project status | `/status` |
| Verify build | `/verify` |
| Single endpoint | `/vertical-slice <endpoint>` |

## Pipeline

```
Idea → Specifier → Contract → [API Planner + Designer] → API Dev → Web Planner → Web Dev → Reviewer → Retro
```

- **Specifier**: Interview-driven spec (asks only what's ambiguous, max 8 questions)
- **Contract**: OpenAPI 3.1 + optional SQL schemas
- **API Planner + Designer**: Run in parallel — task checklist + Figma prompts
- **API Dev**: Implements backend with `@Log()` decorators for structured logging
- **Web Dev**: Implements frontend with LocatorJS for dev tooling
- **Reviewer**: Validates contract alignment, architecture boundaries, logging coverage
- **Retro**: Evaluates pipeline, saves learnings to `.claude/learnings/` for future runs

## Architecture

| Layer | Pattern | Stack |
|-------|---------|-------|
| Backend (default) | Clean Architecture + DDD + DI | TypeScript / Fastify |
| Backend (ML/AI) | Clean Architecture + DDD + DI | Python / Flask |
| Frontend (web) | Feature-Sliced Design | React + Next.js or Expo |
| Frontend (mobile) | Feature-Sliced Design | React Native + Expo |
| Contracts | OpenAPI 3.1 | `contracts/openapi.yaml` |

## Project Structure

```
apps/api/          → TypeScript backend template
apps/api-<name>/   → Python backend (when different stack needed)
apps/web/          → Frontend template (Expo or Next.js)
apps/mobile/       → Mobile frontend (when platform=mobile)
contracts/         → openapi.yaml + db/*.sql
docs/              → spec.md, tasks-*.md, figma.md (generated by agents)
resources/         → brand/, architecture/, figma/

.claude/           → Claude Code agents, skills, commands, rules, hooks, learnings
.github/           → Copilot agents, skills, instructions, prompts
```

## Quick Start

```bash
npm install          # install workspaces
npm run dev          # API on http://localhost:3000
npm run dev:web      # Web on http://localhost:8081
```

Then start an agent:
- **Copilot**: Select `@FastMVP` and describe your idea
- **Claude Code**: Run `/build-mvp your idea here`

## Continuous Improvement

Each pipeline run feeds the **retro agent**, which saves learnings:
- `.claude/learnings/gotchas.md` — accumulated pitfalls and patterns
- `.claude/learnings/metrics.md` — per-run metrics

All agents read gotchas before starting work, creating a **virtuous feedback loop**.

## Non-negotiables

- **OpenAPI is the boundary** — no invented endpoints. Update contracts first, then implement.
- **DDD + DI for all backends** — wiring only in `container.{ts,py}`.
- **Feature-Sliced for all frontends** — API calls only through `shared/lib/api/*`.
- **SQL files for DB** — no migrations, no ORM. Manual execution in DB client.
- **Incremental, not destructive** — features append to spec/contracts/tasks. Never rewrite.
- **`@Log()` decorators** — on all controllers and use-cases. No bare `console.log`.
