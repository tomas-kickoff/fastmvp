FAST MVP
Monorepo MVP Factory · Agentes IA

Fast
MVP

Una fábrica de MVPs completos impulsada por agentes IA especializados — desde una idea en texto plano hasta un backend con DDD, contratos OpenAPI, diseño Figma y frontend funcional, todo en un monorepo.

3 workflows orquestados: FastMVP construye un MVP completo desde cero. Feature Builder agrega funcionalidad incremental sin romper lo existente. Bug Fixer diagnostica y repara bugs con validación automática. Todos comparten la misma arquitectura de agentes y skills.
🏗️
DDD + Clean Architecture
Todos los backends siguen Domain-Driven Design con capas domain → application → infrastructure → interfaces y DI obligatorio. Independiente del lenguaje.
📄
OpenAPI como Contrato
contracts/openapi.yaml es la fuente de verdad entre API y frontend. Nada se inventa: endpoints, schemas y status codes vienen del contrato.
🧩
Feature-Sliced Frontend
Web (Next.js) y Mobile (Expo) usan la misma arquitectura FSD: app / pages / widgets / features / entities / shared. Networking solo vía shared/lib/api/*.
🔌
Skills Dinámicas
Cada agente carga skills según el stack: typescript-api, python-api, react-nextjs-web, react-native-mobile. El mismo API Dev se comporta diferente en TS vs Python.
⚡
Multi-servicio Inteligente
Default: un backend TypeScript/Fastify. Solo se agrega un servicio Python/Flask cuando se necesita un stack diferente (ML, AI). Cada uno con su propio contrato y tareas.
🔄
Incrementalidad
Feature Builder y Bug Fixer operan sobre deltas: leen el estado actual y solo agregan/corrigen lo necesario sin reescribir lo existente.
🚦
Reviewer Automático
Al final de cada pipeline, el Reviewer valida alineación contrato ↔ código, boundaries arquitectónicas y drift. Distingue issues bloqueantes de warnings.
🎨
Integración Figma
El Designer genera prompts para Figma Make. El Design Integrator toma el código exportado y lo adapta: tokens, Feature-Sliced, API boundary. Visual fiel, arquitectura limpia.

Arquitectura multi-agente

Agentes

3 orquestadores que delegan a 8 workers especializados con skills dinámicas según el stack.

🚀
FastMVP
Orquestador · MVP completo
Input
Idea del usuario en texto plano
Delega
Specifier → Contract → API Planner + Designer → API Dev → Web Planner → Web Dev → Reviewer
Output
MVP completo: spec + contracts + API + Web + review
➕
Feature Builder
Orquestador · Feature incremental
Input
Descripción de la feature + estado actual del MVP
Delega
Mismos agentes que FastMVP, pero en modo append/delta
Regla
Nunca reescribe — solo agrega o actualiza quirúrgicamente
🐛
Bug Fixer
Orquestador · Diagnóstico y fix
Input
Reporte de bug del usuario
Delega
API Dev (para bugs complejos de API), Web Dev (para bugs complejos de UI), Reviewer (validación)
Regla
Nunca modifica contracts/openapi*.yaml sin aprobación explícita
📋
Specifier
Worker · Idea → Spec
Rol
Transforma ideas en docs/spec.md canónico. Detecta servicios y plataforma.
Model
Claude Opus 4.6
Output
docs/spec.md (con Open Questions si hay ambigüedad)
📜
Contract
Worker · Spec → OpenAPI + SQL
Rol
Genera contratos OpenAPI 3.1 por servicio + schema SQL opcional.
Model
Claude Opus 4.6
Output
contracts/openapi*.yaml + contracts/db/*.sql
📋
API Planner
Worker · Contracts → Tasks
Rol
Genera checklists ejecutables por servicio: 8–20 tareas con checkboxes.
Model
Gemini 3.1 Pro
Output
docs/tasks-api.md (+ tasks-api-{name}.md por servicio extra)
💻
API Dev
Worker · Implementa backend
Rol
Ejecuta el checklist de tareas y produce código Clean/DDD. TS (Fastify) o Python (Flask).
Model
GPT-5.3-Codex
Regla
Si endpoint falta en contrato: BLOCKED. Actualiza checkboxes conforme avanza.
🎨
Designer
Worker · Figma prompts
Rol
Genera prompts para Figma Make: un master prompt + prompt por pantalla, alineados a OpenAPI.
Model
Claude Sonnet 4.5
Output
docs/figma.md (screen prompts + data mapping table)
📋
Web Planner
Worker · Contracts → Web Tasks
Rol
Genera checklist FSD para frontend: 10–25 tareas. Alineado a OpenAPI y Figma prompts.
Model
Claude Opus 4.6
Output
docs/tasks-web.md
💻
Web Dev
Worker · Implementa frontend
Rol
Ejecuta el checklist web en apps/web/ (Next.js) o apps/mobile/ (Expo). FSD estricto.
Model
GPT-5.3-Codex
Regla
Networking solo vía shared/lib/api/*. No fetch/axios en pages/widgets/shared UI.
🔍
Reviewer
Worker · Validación final (read-only)
Rol
Valida: contrato ↔ API, contrato ↔ Web, boundaries DDD/FSD, drift entre servicios.
Model
GPT-5.3-Codex
Output
Review report: ✅ PASS / ⚠️ WARNINGS / ❌ ISSUES FOUND
